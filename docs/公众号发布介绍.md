# 公众号排版工具 Obsidian 插件 MarkNice-WeChat 开源发布，可一键发送公众号

如果你也用 Obsidian 写东西，又要往公众号上发，大概率遇到过这个流程：笔记写得好好的，复制到公众号编辑器里，标题、引用、代码块全乱了，得手动一行行重新调样式。一篇文章排下来，写作的兴致也磨得差不多了。

MarkNice-WeChat 就是来解决这一步的。它是一个 Obsidian 插件，把笔记转成公众号能直接用的排版，复制粘贴不乱版，配置好之后甚至能跳过编辑器，直接把文章发到公众号草稿箱。今天正式开源。

## 它解决的核心问题

公众号编辑器有个让人头疼的设定：它会丢掉 `<style>` 标签和 CSS class，只认元素自带的 `style` 内联样式。所以普通的 Markdown 转 HTML 工具,粘贴过去样式就掉光了。

MarkNice 的做法是把每一处样式——字号、颜色、间距、边框——都写死进每个元素的 `style` 属性里。这样不管你粘到哪，排版都跟预览里看到的一模一样。

## 主要功能

### 实时预览，所见即所得

打开右侧预览面板，它会按公众号的实际效果渲染当前笔记。你在左边改文字，右边自动刷新。

### 手机 / 桌面双预览

读者大多在手机上看公众号，但你在电脑上排版，两者观感差很多。所以面板加了一个手机模式：把预览裹进一个 375px 的手机画框里，连灵动岛刘海都画上了，让你提前看到读者手机上的真实效果。一键也能切回桌面模式。

### 17 套主题随便挑

内置了 17 套排版主题，从 Claude 暖陶（Anthropic 风格的米白配赤陶色）、经典蓝、杂志衬线，到极客深色、夜空蓝、梦幻紫、海盐青等等。换主题就是下拉框点一下的事，不用碰任何代码。

### 字号和段距能微调

每个人对疏密的偏好不一样。工具栏上有两组加减按钮：字号可以在主题基础上整体 ±6px，段距能在 −16 到 +24px 之间调。关键是这些调整不只是预览好看——复制出去、发草稿时，最终排版完全跟着走。

### 一键复制

点「复制」按钮，到公众号编辑器里直接 `Ctrl/Cmd+V`。本地图片会自动转成 base64 内联进去，粘贴后编辑器会帮你转存。

### 一键发草稿

填好公众号的 AppID 和 AppSecret 之后，可以跳过复制粘贴这一步，直接点「发草稿」。插件会自动走完整个流程：获取凭证、上传封面、把正文里的图片逐张传到微信图床、创建草稿。本地图、`![[wiki 嵌入]]`、远程图片链接都能处理。

### 对 Obsidian 语法友好

`![[图片嵌入]]`、`[[双链]]`、`==高亮==`、Callout、任务列表、表格、代码块这些 Obsidian 常用语法都做了适配。双链会降级成纯文本，代码块保持原样不被破坏。

## 怎么用

插件还没上架社区市场，目前手动安装：

1. 在你的库里建目录 `<vault>/.obsidian/plugins/marknice-wechat/`
2. 把 `main.js`、`manifest.json`、`styles.css` 三个文件放进去
3. 在「设置 → 第三方插件」里启用 MarkNice WeChat

启用后，点左侧栏的报纸图标，或在命令面板执行「打开公众号排版预览」就能开始用了。

如果只是想复制粘贴排版，到这一步就够了。想用一键发草稿的话，再去「设置 → MarkNice WeChat」填上公众号的 AppID 和 AppSecret 即可。AppSecret 只存在本地库的插件数据里，不会上传。

> 两点提醒：
> 公众号接口要求把你的出口 IP 加进白名单（公众平台 → 基本配置），否则会报 40164 错误；草稿箱接口需要已认证的公众号。

## 关于 frontmatter

笔记顶部的 frontmatter 会被当作草稿默认值，省去发布时重复填写：

```yaml
---
title: 文章标题        # 缺省用文件名
author: 作者名         # 缺省用设置里的默认作者
digest: 分享卡片摘要    # 缺省截取正文开头
cover: assets/封面.png # 库内路径或链接，缺省用文中第一张图
---
```
